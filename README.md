# 句子互动 · AI-native 官网原型

把 AI 当**员工**展示，不当功能列表 —— 一个 AI-native 风格的首页重构原型。结果驱动、主动交互、工程文档质感。单文件、零依赖，浏览器直接打开即可。

> 状态：原型（prototype）。页面中的实时指标、对话、城市、员工数据均为**占位示意**，可一键替换为真实数据（见下文「替换真实数据」）。

## 在线预览

启用 GitHub Pages 后访问：`https://guoqichang.github.io/juzi-homepage/`
（Settings → Pages → Deploy from a branch → `main` / `root`）

本地预览：下载 `index.html`，双击用浏览器打开。

## 设计主张

- **AI 当员工，不当功能**：用「我们的团队」替代「7 大产品」卡片矩阵。
- **结果驱动，不是能力驱动**：把跑了一年多的真实业务数据放首页（如转人工率 27% → 2.73%），而不是藏在 case study。
- **主动交互，不做 PPT 式滚动**：页面替访客算账、查、演示，而非读完自己判断。
- **视觉：精确、工程感、不炫技**：克制中性色 + 单一强调色 + 等宽点缀，高信息密度但不拥挤。

## 页面结构

1. **实时状态栏** —— 替代传统 topbar，滚动在岗指标。
2. **Hero 对话窗口** —— 可直接输入体验的 AI 客服对话。
3. **AI 团队墙** —— 每个 AI 员工一张卡，点击查看其能力 / 内部逻辑 / 底层机制 / 方案。
4. **工作台快照** —— AI 员工实际后台界面，对话实时逐条发生，PII 已脱敏。
5. **AI 组织运作（signature）** —— 一笔请求如何穿过这个 AI 组织：渠道 → 路由 → AI 员工 → 闭环，光点实时流动；节点可点开看内部执行轨迹，事件流可点开看请求时间线。
6. **买工具 vs 买结果** —— 结果计算器：访客填自己的情况，当场算出专属对照账本。
7. **实时部署地图** —— 真实经纬度投影的中国地图，城市滚动点亮、闪烁，可点开查看本地业务。
8. **Footer** + **⌘K 命令行式导航**。

## 技术说明

- **单文件**：所有 HTML / CSS / JS 都在 `index.html` 内，无构建、无外部依赖（仅引用 Google Fonts）。
- **动效**：只动 `transform` / `opacity`，并尊重 `prefers-reduced-motion`。
- **地图**：内置真实国界经纬度（Natural Earth / world.geo.json），浏览器内等距投影 + 纬度余弦校正实时绘制。

## 替换真实数据

数据集中在文件顶部的 `CONFIG` 对象，字段名稳定，改这里即可（或后续接口 `fetch` 后填同名字段）：

- `CONFIG.live` —— 状态栏 / Hero 实时指标
- `CONFIG.chatScript` —— Hero 对话脚本
- `CONFIG.team[].detail` —— 员工能力 / 逻辑 / 机制 / 方案
- `CONFIG.workbench` —— 工作台会话
- `CONFIG.cities` —— 部署城市（经纬度 + 本地业务）
- `computeVS()` —— 结果计算器系数
